- **Proposal**: WORKFLOW (one for AGENTS.md ARCH_DECISIONS) — **The auto-fix-vs-manual rule for audit-driven commands.** When a command runs an audit and offers to remediate findings, classify each finding's remediation as `[auto]` only when: (1) the remediation derives from a single canonical source (the HARNESS.md-equivalent for that domain); (2) the remediation is mechanical — same input always produces same output; (3) the remediation requires no user judgement between detection and fix. Multi-source derivations, judgement calls, and operations with user-visible side effects belong in `[manual]` regardless of how trivial they look. Surface them in the drift table, print the suggested command, and let the user trigger them deliberately. The trust boundary should err tighter than feels necessary — auto-fix scope is hard to expand later (users will object) but easy to leave room to grow into. Worth promoting to AGENTS.md ARCH_DECISIONS as a design rule for any future sync-like commands once a second instance recurs.
