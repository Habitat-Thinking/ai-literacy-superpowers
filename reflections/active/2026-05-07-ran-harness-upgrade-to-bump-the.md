The version-consistency CI workflow updated the *badges* automatically (lines 6 and 10) but the *prose-embedded* table row was untouched — this is a class of drift that prose-embedded counts in any markdown surface are vulnerable to, not just README's table.
- **Proposal**: FAILURE (one for `/harness-constrain`) — **README.md marketplace row consistency must be enforced at two timescales.** (1) **PR-time agent check**: extend the `Docs site kept current` constraint scope to include README.md alongside `docs/`, so the harness-enforcer agent verifies on any PR that adds/removes/renames a skill, agent, or command that the README's marketplace plugin table row's version field matches `plugin.json` and the prose-embedded `N skills, M agents, K commands` count matches the directory contents. Both mechanisms are needed: the PR-time check catches drift while context is fresh and the agent can update the prose intelligently; the GC catches anything the PR check missed and fixes mechanical drift cheaply between PRs. This belt-and-braces shape is the same pattern as `Version consistency` (deterministic CI gate at PR time) plus `Plugin manifest currency` (weekly agent GC) — proven on a sibling surface.
